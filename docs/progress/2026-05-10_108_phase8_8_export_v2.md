# 2026-05-10 — Phase 8.8: buildModelZip schemaVersion 2 + animationConfig

[Phase 8 plan](../plan/09_editor_animation_tab.md) 여덟 번째 sprint. baked zip 의 sidecar metadata 에 8.7 IDB 의 animationConfig 를 직렬화해서 Geny 가 자동 적용할 수 있게.

## 변경 surface

### `lib/export/buildModelZip.ts`

- 신규 export `AVATAR_EDITOR_SCHEMA_VERSION = 2 as const` + `AVATAR_EDITOR_SIDECAR_FILE = "avatar-editor.json"` — 두 레포 (geny-avatar, Geny) 의 contract.
- `buildModelZip()` 마지막 단계에 sidecar 직렬화 추가:
  - `loadPuppetAnimationConfig(input.puppetId)` 비동기 로드 (실패 시 warning, export 자체는 진행).
  - sidecar 객체 = `{ schemaVersion: 2, exporter, exportedAt, puppet: {name, runtime, version}, animationConfig?: {...} }`.
  - `JSON.stringify(..., null, 2)` + UTF-8 인코딩 → `avatar-editor.json` 으로 zip 루트에 추가.
- 기존 atlas / model file output 로직 무손상.

### Sidecar schema 잠금

```json
{
  "schemaVersion": 2,
  "exporter": "geny-avatar/live2d",
  "exportedAt": 1730000000000,
  "puppet": { "name": "ellen_joe", "runtime": "live2d", "version": "Cubism 4" },
  "animationConfig": {
    "display": { "kScale": 0.92, "initialXshift": 0, "initialYshift": 0 },
    "idleMotionGroupName": "Idle",
    "emotionMap": { "joy": "red", "anger": "black", "neutral": "tang" },
    "tapMotions": { "HitAreaHead": { "group": "TapHead", "index": 0 } }
  }
}
```

emotionMap 의 값은 expression NAME (인덱스 X). Geny 의 install 이 model3.json 을 파싱해서 NAME → INDEX 변환 (별도 PR — Phase G).

## Backward / forward 호환

- **schemaVersion 1 zip** (Phase A 의 buildModelZip 결과) → Geny C.3 install 디폴트값 사용. 기존 동작 그대로.
- **schemaVersion 2 zip** (본 sprint 결과) → Geny C.3 install 이 puppet 정보만 읽고 animationConfig 무시 (디폴트값). Phase G 가 Geny 를 갱신하면 자동 적용.
- **schemaVersion 3+ zip** (미래) → Geny 가 모르는 버전이면 install endpoint 에서 reject + 명시적 에러 ("Geny upgrade required"). 본 sprint 범위 X.

## 의도적 한계

- **flush 미호출**: ExportButton 의 send-to-Geny 직전에 `usePuppetAnimationConfig.flush()` 호출하면 마지막 슬라이더 변경의 디바운스 (400ms) 보장 가능. 본 sprint 는 안 함 — 사용자가 "send to Geny" 버튼 누르기 전 400ms 사고 시간 가지므로 실제 race 가능성 낮음. 필요 시 후속 hotfix.
- **Spine zip 도 schemaVersion 2 emit**: animationConfig 는 항상 (Spine 도) 직렬화. Spine puppet 의 IDB row 는 비어있어 (animation 탭 자체가 Spine 비활성), `animationConfig` 키 없이 emit 됨. Geny 가 Spine baked import 받을 때는 기존 디폴트 그대로.
- **buildExportZip (raw save) 변경 X**: raw `*.geny-avatar.zip` 은 별도 sidecar 형식 (`avatar.json`) 사용 — 본 8.8 은 baked model zip 만 변경.
- **sidecar 검증 X**: schemaVersion mismatch 같은 client-side 검증 안 함. Geny install 책임.
