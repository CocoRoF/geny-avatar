# 2026-05-10 — Phase 8.6: Hit Areas section (조건부)

[Phase 8 plan](../plan/09_editor_animation_tab.md) 여섯 번째 sprint. model3.json 의 `HitAreas` 가 있을 때만 노출되는 섹션 — 영역 별 tap motion 매핑.

## 변경 surface

### `components/animation/HitAreasSection.tsx` (신규)

- 타입: `TapMotion = { group: string; index: number }`, `TapMotions = Record<HitAreaName, TapMotion>`.
- 한 행 = 한 hit area. 우측 dropdown options 는 모든 motion group × entries 의 cross product (`<group>::<index>` encoded).
- dropdown 변경 시 즉시 `adapter.playMotion(group, index)` 로 미리보기 — 사용자가 어떤 motion 이 매핑됐는지 즉답으로 봄.
- `(none)` sentinel 로 매핑 해제 가능.
- motion 그룹이 0개면 "매핑할 motion 없음" 안내 (드물지만).

### `components/animation/AnimationPanel.tsx`

- `hasHitAreas = meta.hitAreas.length > 0` 가드. 조건 충족 시에만 `<HitAreasSection>` 렌더 — Hiyori / ellen_joe 같은 빈 HitAreas puppet 은 섹션 자체 안 보임.
- 기존 placeholder 섹션 코드 + `hitAreaCount` 변수 정리.

## 의도적 한계

- **value encoding `<group>::<index>`**: native `<select>` 의 단일 string value 한계 회피. nested optgroup 으로도 가능하지만 두 dropdown 으로 분리하면 UX 복잡 — flat list 가 더 명료.
- **multi-motion 한 hit area X**: Geny 의 tapMotions 는 이론상 `{ "HitAreaHead": { "groupA": 1, "groupB": 2 } }` 식 다중 매핑 가능하지만 실제 사용은 단일. 본 sprint 는 단일만 지원, 8.8 export 에서 single-key object 로 직렬화.
- **IDB 영속 X**: 8.7 까지 in-memory.
- **HitAreas 자체 편집 X**: model3.json 의 HitAreas 정의는 read-only — Cubism Editor 영역.
