## Sub-sprint 분할

### Sprint F.1 — Modal layout: sticky actions footer

aside 의 단일 overflow 컬럼 → flex-col 분할:
- 상단 `flex-1 overflow-y-auto`: PROVIDER / MODEL / REGIONS / COMMON CONTEXT / NEGATIVE PROMPT / Active references / Refine / HISTORY / FLOW
- 하단 `flex-none`: [generate] / [apply to atlas] / [reset · keep generating] / [revert texture (F.3)]

scroll 안 가도 항상 actions 보임. 길이 무제한 region 도 동작.

부수 효과: history가 항상 actions 위 scroll 영역에 있어 modal close → reopen 시 자동으로 다시 보임 (이미 IDB persist 되어 있어 데이터는 살아있음).

### Sprint F.2 — Per-region 선택 재생성

각 region tile 에 작은 ↻ (regenerate this region) 버튼. 클릭 시 해당 region 만 OpenAI 호출, 다른 region은 마지막 상태 유지.

state 변경:
- `regionStates: { resultBlob, status, prompt }[]` — 각 region의 현재 결과 추적
- mount 시 각 region의 isolated source canvas 를 초기 resultBlob으로 (즉, 미생성 상태 = 원본 그대로)
- per-region regenerate: 해당 idx만 update
- generate-all: 전체 N개 update (기존 동작)
- 매 update 후 composite 재계산 → RESULT 미리보기 갱신

UI 변경:
- region tile 우측에 ↻ 버튼 + status 표시 (idle/running/succeeded/failed)
- "generate" 메인 버튼 라벨에 호출 횟수 동적 표시: "generate 6 regions" / "generate 1 region"

다른 효과: 비싼 N배 호출이 사용자 통제 하에 들어옴 — 1번만 다시 돌리는 비용은 1번 호출.

### Sprint F.3 — Applied texture override revert 버튼

현재 "reset · keep generating" 은 panel 내 임시 result dismiss 만. 적용된 atlas 텍스처를 원본으로 되돌리는 액션 부재.

- 새 버튼 actions footer 에: "revert texture" (또는 한국어로 "원본 복원" 등)
- click 시 `setLayerTextureOverride(layer.id, null)` → store 갱신 → atlas re-composite → 원본 상태
- confirm dialog 또는 버튼 자체에 destructive 스타일 (빨간 border)
- panel 내 phase / regionStates 도 idle 로 reset
- 적용된 텍스처가 있을 때만 enabled (없으면 disabled with tooltip)

## 의도적 한계

- **F.1 의 height responsiveness**: 90vh modal 안에서 aside 스크롤. 매우 작은 viewport (모바일) 에선 여전히 빡빡할 수 있음. 일단 desktop-first.
- **F.2 의 동시 호출 X**: per-region regenerate 시 다른 region 호출 중이면 deferred? 일단 단순화 — 동시 OK, 각자 자기 status. 충돌 없음.
- **F.2 의 per-region history**: 지금은 history 가 layer 단위. 호출별 / region별 history 는 over-engineering. apply 시 composite 만 history row.
- **F.3 의 undo**: revert 후 "undo revert" 없음. revert 는 단방향. 사용자가 history 에서 revisit → apply 로 복원 가능.
- **F.3 의 mask 영향**: layer mask 는 그대로. revert 는 texture override 만 wipe.
