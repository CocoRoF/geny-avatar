# 2026-05-10 — Phase A.7 (hotfix): Node 22 in Dockerfile

A.1 의 `node:20-alpine` base image 가 Pixi v8 의 SSR 호환성을 깨뜨리던 문제를 hotfix.

## 증상

`docker build` 의 builder stage `pnpm build` 가 `/poc/spine` 정적 prerender 시 `ReferenceError: navigator is not defined` 로 죽음.

```
Error occurred prerendering page "/poc/spine"
ReferenceError: navigator is not defined
    at Object.getNavigator (.next/server/chunks/893.js:38:3636)
```

`/edit/[avatarId]` 같은 dynamic 라우트는 prerender 안 해서 OK. `/poc/spine`, `/poc/cubism`, `/poc/dual` 같은 정적 라우트가 build time 에 server-side 실행되는데, 그 안의 pixi v8 module init 이 `globalThis.navigator.userAgent` 를 read.

## 원인

- 로컬 host (Node 24) 에서 `pnpm build` 통과 — Node 21 부터 `globalThis.navigator` 글로벌 추가됨.
- Docker base image `node:20-alpine` 은 navigator 글로벌 없음 → pixi 가 prerender 시 죽음.
- 즉, Node 21 미만에서는 pixi v8 + Next.js 정적 prerender 조합 자체가 깨짐.

## 변경 surface

### `Dockerfile`

3개 stage 모두 `node:20-alpine` → `node:22-alpine`. runner stage 의 코멘트에 "Node 22 가 필요한 이유" 한 단락 추가 (미래의 누군가가 다시 20 으로 내리지 않도록).

```diff
-FROM node:20-alpine AS deps
+FROM node:22-alpine AS deps
...
-FROM node:20-alpine AS builder
+FROM node:22-alpine AS builder
...
-FROM node:20-alpine AS runner
+FROM node:22-alpine AS runner
+# Why Node 22 and not 20: Pixi.js v8 reads `globalThis.navigator`
+# at module init ... Node 21 added the global; 20 lacks it.
```

### `package.json` — `0.2.1` → `0.2.2`
### `app/page.tsx` — landing chip `v0.2.1` → `v0.2.2`
