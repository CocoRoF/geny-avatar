# 2026-05-09 — E phase Kickoff: 사용자 자율 region 정의

## 컨텍스트

[`64 multi_component_kickoff`](2026-05-09_64_multi_component_kickoff.md) 의 A+B (auto-detect + region-aware UI) 정공 완료. 사용자가 직접 본 그 multi-island layer (上身 = torso + frill 2개) 가 panel UI에서 분리 보이고, region별 prompt + 병렬 호출 + composite 동작.

[`64`](2026-05-09_64_multi_component_kickoff.md) 의 마지막 옵션 E ("사용자가 명시적으로 layer를 sub-region으로 분리"):

> "DecomposeStudio Pro의 한 기능으로, 사용자가 직접 한 layer를 N개 sub-layer로 쪼갬. 이후엔 각 sub-layer가 단일 silhouette이라 기존 파이프라인 그대로 동작. SAM이 들어오면 click으로 island 자동 분리."

A+B 검증 후 사용자 결정: **A+B+E 전부 진행**. E 가 자동 검출 + 수동 override + 영구 저장의 통합 답.

## E 의 가치 (A+B 위에 덧붙는 것)

A+B 만으론 부족한 케이스:

1. **silhouette이 alpha로 연결**되어 있어 한 component로 잡히지만 의미적으로는 분리되어야 함 (예: torso와 어깨가 이어진 atlas, body와 sleeve가 하나로 packed)
2. **silhouette이 의도치 않게 분리**되어 component 가 너무 많이 나옴 (AA edge가 따로 island 등 — 이미 minArea로 어느 정도 필터되지만 한계)
3. **사용자가 region에 의미적 이름 부여하고 싶음** ("torso", "frill", "shoulder") — auto-detect의 "region 1/2/3" 보다 prompt 의도 분배에 도움
4. **명명 + mask가 세션 간에 영구 저장**되어야 함 — 매번 panel 열 때 다시 정의 X

## Sub-sprint 분할

각 atomic PR. 사용자 검증 후 다음 진입.

### Sprint E.1 — Region label 영구 저장 + 명명 UI

가장 가벼운 첫 단. mask 변경 X, auto-detect 결과를 그대로 받되 사용자가 이름만 붙여 영구 보존.

- **IDB v8**: `componentLabels` store. key = `[puppetKey+layerExternalId]`, value = `{ labels: { signature: name } }`
  - signature = `${bbox.x}_${bbox.y}_${bbox.w}_${bbox.h}` (component bbox 기반 stable key)
  - signature는 같은 layer의 같은 source canvas → 같은 component → 같은 bbox → 같은 signature. mask가 안 바뀌면 안정적.
- `useComponentLabels(puppetKey, layerExternalId)` 훅 — `useReferences` / `useVariants` 패턴 재사용
- GeneratePanel REGIONS tile에 inline editable name field (input on blur 저장)
- prompt 조합에 name 사용: `"For region '${name || `region ${idx+1}`}' (1 of 3, ...)"` — 빈 name은 fallback to ordinal

E.1 끝나면: 사용자가 上身 layer에서 region 1을 "torso", region 2를 "shoulder frill" 같이 명명. 다음에 panel 열어도 그대로. region별 prompt가 "torso의 의도" / "frill의 의도" 로 모델에게 전달.

### Sprint E.2 — DecomposeStudio Region split mode (수동 mask 정의)

DecomposeStudio에 새 모드 — 사용자가 직접 region을 paint / SAM-click 으로 정의.

- **IDB v9** (또는 v8 확장): `regionMasks` store. key = `[puppetKey+layerExternalId]`, value = `{ regions: [{id, name, maskBlob}] }`
  - 각 region: name + 사용자가 paint한 binary mask (source canvas dim)
- DecomposeStudio: 새 mode 토글 `paint | erase | regions`
  - regions mode: 좌측에 region list (이름 입력 + 색 매치)
  - 새 region "+" → 색 자동 할당 + name 입력
  - region 선택 후 brush로 paint (해당 region의 mask 추가) / right-click erase
  - **SAM 통합** (Sprint 6.1 backend 활용): "auto-mask here" 클릭 → SAM 호출 → 후보 1~3 → 선택 시 현재 region에 union
- save → IDB 영구 저장

E.2 끝나면: 자동 검출이 부족한 케이스에서 사용자가 SAM 클릭 + 브러시 미세 조정으로 명확한 region 경계 정의. 이름과 함께 영구 저장.

### Sprint E.3 — GeneratePanel 통합 (manual region 우선, auto fallback)

E.2 의 manual region 이 있으면 우선 사용, 없으면 auto-detect 로 fallback.

- GeneratePanel mount 시 `useRegionMasks(puppetKey, layerExternalId)` 호출
- manual regions 있으면:
  - `findAlphaComponents` 대신 manual region masks 그대로 사용
  - 각 region name = 사용자 정의
  - `prepareOpenAISourcesPerComponent` 흐름 그대로 (mask가 다른 출처일 뿐)
  - REGIONS tile에 "manual" 표시 + DecomposeStudio 열기 link
- manual regions 없으면: 현재 A+B 동작 그대로 (auto-detect, E.1의 label 활용)
- 진단 로그에 region 출처 (auto / manual) 표시

E.3 끝나면: 사용자 의도 → DecomposeStudio 에서 region 정의 → GeneratePanel 에서 region별 prompt → 정확한 결과. 자동 검출이 충분한 layer는 그대로 자동, 부족한 layer만 사용자가 명시.

## 의도적 한계

- **E.1 의 signature 안정성**: bbox 기반 키는 layer source 이미지가 약간만 변해도 (1px 차이) 다른 키가 됨. 그땐 이름이 사라짐. 사용자가 다시 입력. layer의 atlas region이 큰 폭으로 변하지 않는 일반 경우엔 충분히 안정.
- **E.2 의 region overlap**: 두 region이 겹치는 경우는 정의상 단일 component 로 합쳐서 다루는 게 맞음. UI에서 overlap 경고만 표시 + 동작은 union으로 진행.
- **E.2 의 brush 위에 SAM**: SAM mask가 brush 영역과 합쳐질 때 union / replace 선택 — 일단 union 만, 더 정교한 boolean은 후속 Sprint 6.3 (multi-mask boolean composition) 로 미룸.
- **Export/Import 영향**: regionMasks 가 IDB-only 인 사이엔 export ZIP 에 안 들어감. 다른 PC 에서 같은 puppet 받아도 region 재정의 필요. export 통합은 후속.
- **Phase 6 SAM 의 UI 통합**: Sprint 6.2 가 미진행 상태인데 E.2 가 SAM 통합을 깔아둠. 6.2의 segment mode가 사실상 E.2 에서 자연 통합됨 — Phase 6 sprint 표 갱신 가능.

## 예상 산출물

E.1~E.3 끝나면:
- 사용자가 layer마다 의미 있는 region 이름 부여 (영구)
- 자동 검출이 모호한 layer는 SAM 클릭 + brush 로 직접 region 정의
- GeneratePanel 이 manual / auto 구별 없이 동일한 region-aware UI 제공
- 모델에게 "torso 부분", "shoulder frill 부분" 처럼 의미 있는 분배

A+B+E 합쳐서 multi-region layer의 generation 정밀도 끝까지 끌어올림.
