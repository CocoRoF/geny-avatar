## Cubism 디버깅에서 가져갈 학습

1. **옵셔널 체이닝(`a?.b?.()`) 호출은 silently no-op**. 어댑터 경계에서 외부 라이브러리 메서드를 부를 때 그 메서드가 실제로 존재하는지 진단 로그로 확인. 11~15라운드의 `setPartOpacity` 호출은 모두 위약이었음 — 16에서 진단 로그 박은 한 줄 (`hasSetPartOpacity=false`)로 즉시 드러남.
2. **엔진의 update 사이클을 가정으로 추측하지 말 것**. d.ts에 `internalModel.update(dt, now)` 한 메서드로 전체 사이클이 명시돼 있었고, 이벤트(`beforeModelUpdate`)는 그 안의 잘못된 시점이었음. monkey-patch로 원본 호출 후 mutate가 정답.
3. **Cubism Live2D는 모델별 part-drawable binding이 다름**. setPartOpacity가 작동하는 모델·작동 안 하는 모델 둘 다 있음. drawable opacity Float32Array 직접 mutation은 모델 무관하게 작동.
4. **Pixi/엔진 라이브러리는 native typed array view를 노출함**. wrapper 메서드가 사본 반환하는 경우가 흔하므로, mutate가 필요할 때는 native handle (`coreModel._model.drawables.opacities`) 우선.
