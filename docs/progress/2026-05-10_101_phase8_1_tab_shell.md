# 2026-05-10 — Phase 8.1: Editor Tab Switcher Shell

[Phase 8 plan](../plan/09_editor_animation_tab.md) 의 첫 sprint. 두 editor 페이지 (`/edit/[avatarId]`, `/edit/builtin/[key]`) 헤더에 `[edit | animation]` 탭 chip 추가, URL 쿼리 (`?tab=animation`) 기반 state. Animation 탭은 placeholder — 8.2~8.7 이 채움.

## 변경 surface

### `components/animation/EditorTabBar.tsx` (신규)

- `useEditorTab()` hook — `useSearchParams()` 로 `?tab=` 읽음. `"animation"` 외에는 `"edit"` 로 fallback.
- `<EditorTabBar />` — 두 chip pair. `router.replace(...)` 로 history 안 더럽힘. `?tab=` 없으면 query 자체 삭제 (URL 깔끔).

### `components/animation/AnimationPanel.tsx` (신규)

- props: `{ puppetKey, adapter }`. 향후 sprint 들이 puppetKey 로 IDB 접근, adapter 로 motion/expression 트리거.
- `adapter.runtime === "live2d"` 가드 — Spine puppet 진입 시 "V1 범위 밖" placeholder (Cubism motion/expression 과 Spine animation tracks/skins 가 다른 모델이라).
- 4 섹션 placeholder (display / motions / expressions / hit areas) — 각각 다음 sprint 번호 표기.

### `app/edit/[avatarId]/page.tsx`

- `useEditorTab()` 호출.
- 헤더의 `<span>edit</span>` 장식을 `<EditorTabBar />` 로 교체.
- `<aside>` 안: `activeTab === "edit"` ? 기존 panels (Tools / References / Variants / Layers) : `<AnimationPanel>`.

### `app/edit/builtin/[key]/page.tsx`

- 동일 패턴. 헤더의 `edit · builtin` chip 은 `<EditorTabBar /> + builtin` 으로 분리 (탭 전환 후에도 builtin 표시 유지).

## 의도적 한계

- **AnimationPanel 모든 섹션이 placeholder**: 의도. 본 sprint 는 shell + 라우팅만. 8.3~8.6 이 실제 UI 채움.
- **Spine puppet 의 Animation 탭 진입 가능**: 안내 메시지만 노출. Edit 탭 / Send to Geny 는 정상 사용 가능 — Geny 측이 Spine 메타 디폴트 적용.
- **탭 transition 애니메이션 X**: instant swap. 캔버스는 양쪽 탭에서 mount 유지라 puppet 깜빡임 없음.
- **shortcut 단축키 X**: `?` 같은 키로 탭 전환 단축은 후속. 일단 click only.
