## 의도적 한계

- **playing chip 의 inaccuracy**: 4초 fixed timer. 짧은 motion (~1s) 도 4초 동안 chip 떠있음. 긴 motion (>4s) 은 chip 일찍 사라짐. 정확한 추적은 engine internal API (motionManager 의 isFinished 등) 이용해야 하는데 unstable. 현재 trade-off 수용.
- **motion-end 후 자동 idle 복귀**: 엔진의 default 동작. 따로 안 건드림.
- **idle group 변경의 실제 적용 X**: 8.3 의 dropdown 은 state 만 저장. PuppetCanvas mount 시 자동재생되는 idle 은 여전히 manifest 의 첫 그룹 / "Idle" 이름. 사용자가 명시적으로 idle 그룹의 ▶ 클릭하면 됨.
- **motion3.json duration 미표시**: 본 sprint 는 filename basename 만. 8.2 의 cubismMeta 가 fadeIn/fadeOut 만 추출 — duration 은 motion3.json 의 별도 필드라 해당 파일까지 추가 fetch 필요. 추후.
- **Spine motion 미지원**: AnimationPanel 의 isLive2D 가드가 먼저 잡음.
