# 2026-05-10 — Phase 8.7: IDB persistence for animation config

[Phase 8 plan](../plan/09_editor_animation_tab.md) 일곱 번째 sprint. UI 레이어 (8.1~8.6) 의 in-memory state 들을 한 IDB store 로 영속화 + 단일 source of truth 로 합침.

## 변경 surface

### `lib/persistence/db.ts`

- 신규 type `PuppetAnimationConfigRow` — `{puppetKey, display, idleMotionGroupName, emotionMap, tapMotions, updatedAt}`. 한 puppet 당 한 row.
- Dexie schema bump v9 → v10 with new store `puppetAnimationConfig: "puppetKey, updatedAt"`. 기존 store schema 무손상.
- 추가된 row CRUD:
  - `loadPuppetAnimationConfig(puppetKey): Promise<row | null>`
  - `savePuppetAnimationConfig({puppetKey, ...value})` — `updatedAt` 자동.
  - `deletePuppetAnimationConfig(puppetKey)` — 향후 puppet 삭제 cascade 용 (현재 puppet delete 는 다른 store cascade 까지만).

### `lib/avatar/usePuppetAnimationConfig.ts` (신규)

`{ config, loading, update, flush }` 반환:

- **load**: mount 시 + puppetKey 변경 시 IDB read (cancel 가드 포함). 행 없으면 DEFAULTS.
- **update(patch)**: 부분 갱신 + 400ms debounced save. `setTimeout` 안에서 `configRef.current` 사용해서 마지막 patch 까지 반영된 상태 저장. 디바운스 중 호출이 누적되면 한 번의 save 로 collapse.
- **flush()**: pending 디바운스 강제 즉시 flush. 8.8 의 "send to Geny" 직전에 호출 예정.
- **unmount cleanup**: 펜딩 timer 있으면 즉시 한 번 fire (사용자가 슬라이더 만지자마자 탭 닫는 케이스 보호).
- `puppetKey === null` short-circuit — 다른 puppet-scoped 훅과 동일 패턴.

### `components/animation/AnimationPanel.tsx`

- `usePuppetAnimationConfig(puppetKey)` 호출.
- `metaLoading` (8.2) + `configLoading` (8.7) 둘 다 false 일 때만 sections 렌더 — `<DisplaySection initial={...}/>`, etc. 에 IDB-loaded 값 inject.
- sections 의 `initial` prop 이 lazy `useState` 초기화에 들어가서 `key={puppetKey}` 로 puppet 전환 시 강제 remount → 새 puppet 의 IDB 값 적용.
- 4개 onChange 콜백 — 받은 patch 를 `update({...})` 로 hook 에 전달.
- placeholder Section helper 제거 (사용처 없어짐).

## 의도적 한계

- **section 의 controlled 화 X**: 8.6 까지 self-managed state 그대로. 그 결과 sections 의 IDB 값 → UI 반영은 mount 시점에만 일어남 (나중 IDB 변경은 self-state 가 우선). 일반 사용에선 한 puppet 에 대해 한 사용자가 만지므로 문제 X.
- **emotionMap 의 타입 호환**: hook 의 row.emotionMap 은 `Record<string, string>`, ExpressionsSection 의 EmotionMap 은 `Partial<Record<EmotionKey, string>>`. cast 로 처리 — 실제 키는 동일 셋이라 안전.
- **다른 store 와의 cascade X**: deletePuppet 시 puppetAnimationConfig row 자동 삭제 안 됨 (별도 sprint 필요). hobby 사용에선 puppet delete 자체 드물어 우선순위 낮음.
- **migration 데이터 X**: v9 → v10 은 신 store 추가만, 기존 row 변환 없음. 이미 사용 중인 IDB 도 무손상.
- **flush() 미사용 (현재)**: 8.8 의 export 흐름이 호출. 8.7 만 보면 dead code 처럼 보이지만 의도된 사전 작업.
