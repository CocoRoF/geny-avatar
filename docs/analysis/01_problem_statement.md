# 01 — Problem Statement

## 우리가 만들려는 것

**웹 기반 2D Live Avatar 편집기.** 사용자는:

1. 무료로 공개된 **뼈대(rigged puppet)** 샘플을 불러온다.
2. 화면 좌측에서 캐릭터/프리셋을 고르고, **중앙에서 라이브 미리보기**(애니메이션 가능)를 본다.
3. **우측의 레이어 패널**에서 개별 부위(머리카락 한 가닥, 모자, 무기, 옷의 한 조각 등)를 보이기/숨기기·색조·투명도 조정.
4. 한 레이어를 선택하고 "AI로 이 텍스처를 다시 만들어줘" 또는 "이 모자를 베레모에서 카우보이햇으로 바꿔줘"라고 지시하면 **텍스처가 그 슬롯에 그대로 입혀진** 채로 미리보기가 갱신된다.
5. 결과를 다시 패키징해서 내보낸다 (자체 포맷 + 원본 런타임 호환 atlas/모델).

UI 톤의 레퍼런스는 [nikke-db.pages.dev/visualiser](https://nikke-db.pages.dev/visualiser). 좌측 캐릭터 리스트, 중앙 프리뷰, 우측 도구·레이어 에디터.

## 핵심 차별점

- NIKKE visualiser는 **뷰어**다 (Spine 4.0/4.1을 그대로 보여줌, 색상·투명도 토글만).
- 우리는 **편집기 + 텍스처 생성기**다. 레이어를 보는 것을 넘어 **다시 그린다**.
- 생성형 AI를 통해 "이 옷의 텍스처를 새로 만든다"가 일상적인 조작이 되는 것이 목표.

## 왜 어려운가 — 진짜 문제 세 가지

### 문제 1 — 좋은 뼈대 + 깔끔한 레이어를 가진 무료 자산이 흔치 않다

공개된 무료 puppet은 (a) 라이선스가 제한적이거나 (Live2D 공식 샘플은 작은 사용자만 상업 가능), (b) 텍스처 atlas가 부위 단위로 깨끗하게 나뉘어 있지 않거나, (c) 베이크된 그림자/반사로 인해 한 슬롯의 텍스처에 다른 부위의 잔흔이 묻어 있다.

→ **자산을 받아서 그대로 쓰는 게 아니라, 받아서 "정제"해야 한다**는 뜻이다. 이 정제 자체가 도구의 일부가 되어야 한다.

### 문제 2 — Atlas는 픽셀 격자, 슬롯은 의미 단위 — 사이의 매핑이 깨진다

Spine `.atlas` / Live2D 텍스처는 패킹 효율을 위해 여러 부위를 **한 PNG에 압축**한다. 슬롯/드로어블 입장에서는 "내 영역은 이 사각형"이라는 UV가 있을 뿐이지만, **그 사각형 안에 다른 부위의 잔상이 같이 그려져 있는 경우**가 흔하다 (예: 머리카락 안쪽 면에 피부 음영이 같이 들어가 있음).

→ **레이어 단위 텍스처를 진짜로 분리해야 AI가 "이 모자만"을 다시 그릴 수 있다.** 분리는 자동(SAM·세그멘테이션)·수동(브러시 마스크)·반자동(AI 추정 후 사용자 확정)의 조합이 필요하다.

### 문제 3 — AI 생성물이 슬롯의 UV·외곽선·해상도를 지켜야 한다

생성형 AI에게 "베레모 텍스처 그려줘"라고 그냥 시키면 외곽선이 슬롯의 UV 사각형과 어긋난다. 새 텍스처를 atlas에 다시 끼워넣었을 때 모양이 안 맞으면 미리보기가 깨진다.

→ **마스크·외곽선·기준 해상도를 강제하는 inpainting/ControlNet 워크플로우**가 필수다. 단순한 "이미지 생성"이 아니라 **제약 조건이 있는 텍스처 합성**이다.

## 우리가 명시적으로 **하지 않는** 것 (스코프 컷)

- 새 puppet을 처음부터 리깅하지 않는다. 리깅은 Inochi Creator·Live2D Cubism Editor·Spine Editor 같은 전용 데스크톱 도구의 영역이다. 우리는 이미 리깅된 puppet을 **편집·재텍스처**한다.
- 페이스 트래킹(VTubing 라이브 캠 입력)을 하지 않는다. 미리보기는 사전 정의 애니메이션 또는 파라미터 슬라이더로 충분하다.
- 3D 아바타(VRoid, VRM)를 다루지 않는다. 2D rig 전용.
- 음성·립싱크·오디오 입력을 다루지 않는다 — UI에 있던 "Yap mode" 같은 것. 미래 작업.

## 성공 기준 (MVP)

이 셋이 다 되면 MVP 끝:

1. 무료 샘플 puppet 1종(예: Live2D Hiyori 또는 Spine spineboy 변형)을 불러와 중앙에 살아있는 미리보기로 띄운다.
2. 우측 레이어 패널에서 슬롯/드로어블을 토글·색조 조정할 수 있다.
3. 슬롯 하나를 선택하고 텍스트 프롬프트를 주면, AI가 그 슬롯의 텍스처를 inpaint해서 미리보기가 즉시 갱신된다 — 외곽선이 어긋나지 않은 상태로.

성공 기준이 되는 순간 = "이걸 다른 사람에게 보여줬을 때 '오 이거 흥미로운데'라고 반응할 수 있는 상태."
