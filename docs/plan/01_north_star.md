# 01 — North Star

V1을 한 문장으로 그린다. 이 문장이 맞으면 끝, 틀리면 모든 실행을 다시 잡는다.

## V1 한 문장

> **무료 puppet을 받아 웹에서 띄우고, 레이어별로 보고/숨기고/색을 바꾸고, AI로 텍스처를 다시 그려서 같은 외곽선에 입힌 결과를 라이브 미리보기로 즉시 볼 수 있다.**

## V1을 자르는 5가지 능력

1. **Bring** — 사용자가 puppet 패키지를 가져온다. 두 경로 모두 day-1:
   - 내장 샘플 (Live2D 공식 1종 + Spine 공식 1종 — 클릭으로 즉시 로드)
   - **사용자 업로드** (드래그-드롭 ZIP 또는 파일 폴더). Spine `.skel/.json + .atlas + .png` 또는 Cubism `.model3.json + .moc3 + 부속`. 포맷·버전 자동 감지.
2. **See**: 살아있는 미리보기. 애니메이션이 돌고 파라미터를 슬라이더로 조작 가능. **두 포맷 모두 같은 미리보기 캔버스에서 동등하게 작동**.
3. **Inspect**: 모든 레이어를 한 패널에서 본다. 검색·bulk select·visibility/color 토글. Spine slot이든 Cubism drawable이든 같은 UI.
4. **Decompose**: 한 레이어의 텍스처 region을 atlas에서 추출한다. 깨끗한 자산이면 그대로, baked-in이면 마스킹 보조.
5. **Regenerate**: 한 레이어를 선택하고 프롬프트(+옵션 ref 이미지) → AI가 같은 외곽선에 새 텍스처 → 미리보기 즉시 갱신.

다섯이 다 되면 V1. 하나라도 빠지면 V1이 아니다. **두 포맷 중 하나만 작동해도 V1이 아니다.**

## V1 데모 시나리오 (수락 기준)

**시나리오 A — 인터넷에서 받은 Live2D 모델 업로드**:
1. 사용자가 nizima 또는 shiralive2d에서 받은 Cubism 모델 ZIP을 드래그-드롭.
2. 시스템이 `.model3.json`을 자동 감지, Cubism 4 모델로 인식.
3. 미리보기에 정상 출력 + idle motion 자동 재생.
4. 우측 레이어 패널에서 "옷 상의" 선택, "Generate" 클릭, "white tank top with red ribbon" 입력.
5. 30초 안에 미리보기가 새 텍스처로 갱신. 외곽선 어긋남 없음.
6. "Export" → ZIP에 갱신된 텍스처 + 메타데이터.

**시나리오 B — 인터넷에서 받은 Spine 자산 업로드**:
1. 사용자가 Spine 4.1 puppet (`.skel + .atlas + 페이지 PNG들`) 폴더를 드래그-드롭.
2. 시스템이 `.atlas`와 `.skel`을 감지, Spine 4.1로 인식. 같은 UI에서 Live2D와 동일하게 표시.
3. 한 슬롯의 attachment를 선택, baked detail이 있어서 Auto-decompose 결과가 부정확.
4. 사용자가 브러시 도구로 마스크 다듬기.
5. "Regenerate" → AI가 마스크 영역만 새로 그림 + 외곽선 보존.
6. 결과 만족 → 적용.

**시나리오 C — 두 포맷 같은 세션에서 자유롭게**:
1. 같은 세션에서 Live2D 모델로 작업하다가 Spine 모델로 전환.
2. UI가 동일한 패널 구조로 응답 (capability flag로 일부 버튼만 활성/비활성).
3. 두 작업 결과 모두 IndexedDB에 보존, 새로고침 후에도 복원.

(자산 출처는 상태바에 정보용 chip로만 표시. 차단·동의 흐름 없음.)

## V1이 **아닌 것** (스코프 컷, 명시적)

- 페이스 트래킹 / 라이브 카메라 입력 (VTubing).
- 음성·립싱크·오디오.
- 3D 아바타 (VRM / VRoid).
- puppet 처음부터 리깅 / 본 추가.
- 캐릭터 LoRA 학습 UI (외부에서 학습한 LoRA만 받는다).
- 공유 갤러리 / 멀티유저 / 협업 편집.
- 모바일 first (반응형은 P3 이후).

## 사용자

V1의 1차 사용자는 **인터넷에서 puppet을 받아 가지고 노는 사람** — 자기 puppet을 가진 일러스트레이터, NIKKE/원신/블루아카이브 자산을 손에 넣은 fan, Live2D Hub에서 무료 모델을 받은 호기심 많은 개발자. 일반인 대중이 아니다 (그건 V2 이후).

## 성공 측정

V1이 성공했다고 말하려면:
- 시나리오 A (Live2D 업로드), B (Spine 업로드), C (포맷 전환)를 모두 영상으로 시연 가능
- 첫 페인트(빌드인 모델 떠오르는 시간) 1초 내
- 사용자 업로드 → 첫 미리보기 5초 내 (50MB 미만 자산 기준)
- 텍스처 재생성 라운드트립 30초 내 (1080p 데스크톱 + Replicate)
- 한 puppet으로 export → 다시 import → 동일 결과 재현 (라운드트립 안정성)
- 인터넷에 흔한 자산 5종 무작위 테스트(2 Spine + 3 Live2D, 다양한 버전) 시 80% 이상 정상 로드

## 안 봐도 되는 것

V1 단계에서는:
- 성능 최적화는 "동작하면 OK". P3 이후에 다듬는다.
- 디자인은 "볼만하면 OK". 사용자가 늘어난 뒤 디자이너 합류.
- 다국어 — 한국어 우선, 영어는 P2 이후.
