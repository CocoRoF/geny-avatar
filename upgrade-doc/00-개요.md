# upgrade-doc — 전면 검수 + 강력한 텍스처 에디터 + 전신 리스타일

2026-06-10 시작. 이 폴더는 세 가지 목표를 위한 **종합 진단과 실행 계획**이다.

## 목표 (사용자 요구사항 그대로)

- **G1 — 전체 기능 검수 + 완벽 개선.** 현재 에디터든 Gen(AI 생성)이든 문제가 있다.
  코드 전수 감사로 결함을 모두 찾아내고 수정한다.
- **G2 — 강력한 텍스처 에디터.** Cubism Editor 급은 아니어도, 텍스처를
  간단하게라도 확실히 수정할 수 있는 포토샵류 에디터로 끌어올린다.
- **G3 — 레퍼런스 기반 전신 리스타일.** 레퍼런스가 제공되면 그것을 바탕으로
  특정 부분을 정확하게 수정하고, 나아가 **개별 part 단위가 아니라 전체
  이미지(아틀라스 페이지) 단위로 텍스처를 통째로 바꿀 수 있는** 흐름을 만든다.

## 불변 제약 (어기면 머지 금지 — docs-upgrade/plan/00 의 5원칙 승계)

1. **하위 호환 절대 불파괴** — 기존 puppet 파일·IndexedDB·세션은 업그레이드 후에도 그대로 열린다.
2. **per-layer 파이프라인 유지** — 새 기능은 기존 `submitGenerate` + `setLayerTextureOverride` 위에 얹는 레이어다. 페이지 오버라이드도 기존 레이어 오버라이드와 **합성 순서로 공존**한다 (대체 아님).
3. **기존 기능 회귀 금지** — 매 단계마다 기존 흐름(업로드→렌더→마스크→생성→적용→export)이 그대로 동작해야 한다.
4. `pnpm typecheck` + `pnpm lint` 통과가 커밋 조건.
5. progress 로그는 docs-upgrade/progress 의 양식을 따라 **이 폴더의 `progress/`**에 append-only 로 누적한다.

## 기존 문서와의 관계

- `docs/` — V1 (Phase 0~8 + Phase A) 기록. 완료된 과거.
- `docs-upgrade/` — "whole-character generative editor" 분석/계획.
  **Phase 1 + Phase 1.x(inpaint/mask 대수술)까지만 구현**되었고 Phase 2/3/4 는 미착수.
  그 분석(특히 08-recommended-architecture, 13-failure-modes)은 여전히 유효한
  배경 지식이며, 이 폴더의 계획은 그 위에서 **현재 코드 현실**에 맞게 다시 세운 것이다.
- README/docs 의 일부 기술 서술은 코드와 어긋난다 (예: dead code 인 router,
  존재하지 않는 maskConvert.ts 참조). **코드가 1차 출처다.**
