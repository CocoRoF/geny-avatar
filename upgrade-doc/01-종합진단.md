# 01 — 종합 진단 (2026-06-10 코드 기준)

4개 서브시스템 전수 감사 결과의 아키텍처 맵. 결함 자체는 [02-결함목록.md](02-결함목록.md).

## 1. 렌더링 / 어댑터 계층

### 로드 경로
1. `PuppetCanvas` (components/PuppetCanvas.tsx:42) → `usePuppet` (lib/avatar/usePuppet.ts:45)
   가 Pixi `Application` 생성 → `createAdapter` (lib/adapters/AvatarRegistry.ts:23).
2. **Live2D**: `Live2DAdapter.load` (lib/adapters/Live2DAdapter.ts:149) —
   Cubism Core 대기 → 텍스처 preload(blob: URL 대응 강제 loadParser) → cdi3(표시명/그룹)
   → pose3(bakedHidden) → `Live2DModel.from` → part→drawable 매핑 + 텍스처 페이지 카탈로그
   (`textureSourcesById` 에 **pristine 비트맵 고정 보관**) → **(part × atlas page) 당 Layer 1개**
   (멀티페이지 파트는 `#p{N}` externalId suffix) → `internalModel.update` 몽키패치로
   매 프레임 오버라이드 적용 (:462–481, :556–597).
3. **Spine**: `SpineAdapter.load` (lib/adapters/SpineAdapter.ts:68) — slot 당 Layer 1개,
   atlas region 의 `rotated` 플래그 유지.
4. 스토어: `useEditorStore.setAvatar` (lib/store/editor.ts:102) 가 layer 기본 가시성 시드.

### 텍스처 오버라이드 파이프라인 (핵심)
- 단위: **Layer = Cubism (part×page) / Spine slot**. 오버라이드 blob 은 layer 의
  upright bbox rect (`TextureSlice.rect`) 크기 PNG.
- 흐름: DecomposeStudio/GeneratePanel → `editorStore.layerMasks / layerTextureOverrides`
  → `useLayerOverridesPersistence` 가 IDB `layerOverrides` 미러 (키: puppetKey + layerExternalId + kind)
  → **GPU 적용 지점은 LayersPanel 의 effect 단 한 곳** (components/LayersPanel.tsx:74–83)
  → `applyLayerOverrides` (lib/adapters/applyOverrides.ts:35–96).
- `applyLayerOverrides` 동작: 매 호출마다 **모든 페이지**를 pristine 비트맵에서 새 캔버스로
  재구성 → dirty 페이지에 texture(트라이앵글 클립 + destination-out 와이프 후 source-over)
  → mask(destination-out, 클립 없음) → 모든 페이지 `replacePixiTextureSource` 로 GPU 재업로드.
- **페이지 전체 교체 API 는 없음** — 메커니즘은 이미 페이지 단위인데 (페이지 재구성 + 페이지 스왑)
  베이스가 pristine 으로 하드코딩 (applyOverrides.ts:62). `Texture.origin:"override"` 도메인
  필드는 존재하나 미사용 (lib/avatar/types.ts:109–116). → G3 의 기반공사 지점.

### 읽기 경로
- 썸네일/스튜디오 소스: `extractLayerCanvas`/`extractCurrentLayerCanvas`
  (lib/avatar/regionExtract.ts:106–198) — pristine 페이지에서 rect 크롭 + 트라이앵글 클립.

## 2. Decompose Studio (에디터)

### 모드와 저장 대상
| 모드 | 출력 | 저장 |
|---|---|---|
| Mask | RGB black + alpha = hide 양 (threshold 베이크) | `editorStore.layerMasks` → IDB |
| Mask(embedded, GeneratePanel MASK 탭) | RGB white = inpaint 영역 | 부모 콜백 (스토어 미경유) |
| Split | region 별 binary mask PNG | IDB `regionMasks` |
| Paint | 텍스처 직접 페인팅 결과 PNG | `layerTextureOverrides` → IDB |

### 도구
- StrokeEngine (lib/avatar/decompose/strokeEngine.ts): spacing 보간 dab, pen pressure, hardness
  gradient — 품질 좋음.
- Bucket/Wand: worker 기반 flood fill (floodFillClient/Worker), wand 는 compose op(add/sub/intersect)
  + morphology(grow/shrink/feather/invert).
- SAM: split 모드 전용, `/api/ai/sam` (Replicate sam-2).
- History: 변경 캔버스 전체 스냅샷 ≤30개 (useHistory.ts) — 메모리 상한 없음.
- 합성 표시: WebGL2 셰이더 컴포지터 + Canvas2D fallback (compositor.ts).

### Paint 모드 현재 깊이
브러시/지우개/스포이드 코어는 견고. 그러나 불투명도 슬라이더 없음, 직선(Shift) 없음,
블렌드 모드 없음, 스와치 없음, bucket/wand 가 페인트 결과가 아닌 **원본 소스를 참조**(결함),
지우개 결과가 재진입 시 유실(결함). → G2 에서 강화.

## 3. AI 생성 파이프라인

### 흐름 (이미지 수학은 전부 클라이언트)
1. 패널 마운트: 소스 3종 추출(AI용 base+override / pristine / preview) + region 검출
   (수동 region 우선, 없으면 connected components) + OpenAI 1024² 패키지 사전 준비.
2. submit: provider 별 소스/마스크 prep 분기 → (옵션) refine-prompt (vision chat)
   → canonical-pose 스냅샷 (OpenAI 전용, refs ≤3 일 때) → multipart POST `/api/ai/generate`.
3. 서버: `createJob` 후 **fire-and-forget** `void runJob` (app/api/ai/generate/route.ts:105).
   job 은 `globalThis` 인메모리 Map (lib/ai/server/jobs.ts) — 영속 없음, TTL 1h, 프루닝은
   createJob 안에서만.
4. 클라: 1.5s 폴링 ≤300s → result blob → postprocess (패딩 크롭 + 리샘플 + alpha-enforce
   + erosion 2–8px) → blend 모드(ai-only/hard/soft) → apply 시 `setLayerTextureOverride`
   + IDB `aiJobs` 히스토리.

### Provider 현실
- **OpenAI gpt-image**: 주력. multi-image edits, mask 는 hard mask 가 아니라 **soft reference
  hint** (image[] 동봉 + 프롬프트) — 모델측 마스크 신뢰 불가가 결론이라 클라 blend 가 최종 보장.
- **fal FLUX 계열**: atlas-crop 용도로는 아키텍처 한계 확정 (docs-upgrade/progress/2026-05-13-mask-as-reference-hint.md). default 가 quality 낮은 flux-pro-fill 로 남아있음.
- **Gemini**: mask 의미론이 hide(반전) 로 다름, refs 미지원.
- **Replicate**: generate() 가 항상 throw 하는 stub 인데 picker 에 노출됨.
- **Dead code**: `lib/ai/router.ts` (미호출), `buildOpenAIEditMask`, `convertInpaintMaskToOpenAIPadded`,
  존재하지 않는 `maskConvert.ts` 를 가리키는 주석 2곳.

### 레퍼런스 현재 동작
- puppet 단위 IDB 저장 (`puppetReferences`), 제출 시 image[] 뒤에 동승. 영향은 **프롬프트 매개**
  (refine LLM 이 ref 의 해당 부위 묘사를 풀어쓰는 방식). IP-Adapter 류 조건화 없음.
- 스코프는 철저히 per-layer/per-region. 전신 일관 리스타일 오케스트레이션 없음 → G3.

## 4. 데이터 / 영속 / Export

### Dexie (DB `geny-avatar`, v1–v10, 10개 테이블)
puppets, puppetFiles, aiJobs, variants, layerOverrides(kind: mask|texture),
puppetSessions, puppetReferences, componentLabels, regionMasks, puppetAnimationConfig.
- 키 규약: 런타임 재생성되는 `Layer.id`/`TextureId` 대신 **puppetKey + layerExternalId**.
  페이지 단위 확장 시 안정 키는 **pageIndex** (TextureId 는 로드마다 재발급 — 영속 키 금지).

### Export 3종
- `bakeAtlas`: pristine → texture(트라이앵글 클립) → mask(클립 없음 — 결함) 순 합성.
- `buildModelZip`: 페이지 PNG 교체 + 숨김 파트 pose3/motion3 패치 + `avatar-editor.json` sidecar.
- `buildBundle` (geny-avatar.zip): bundle + overrides + LICENSE. **유실**: aiJobs 히스토리,
  references, regionMasks/labels, **animationConfig**(model zip sidecar 에만 있음), puppet id.

### Geny 연동
IDB 쓰기 → `_triggerSyncPush` → 600ms 디바운스 → bake → `POST /api/library/sync` → 공유 볼륨.
열기만 해도 썸네일 갱신 → re-publish churn (결함).
