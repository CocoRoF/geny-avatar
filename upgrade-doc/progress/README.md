# progress — work log (append-only)

Same operating rules as docs-upgrade/progress: one file per work unit (≈ PR),
`YYYY-MM-DD-title.md`, no after-the-fact edits (correct via a new entry). Format:

- **What** — defect ID / plan-document reference
- **How** — the key decision and code location
- **Verification** — typecheck/lint + manual checks
- **Left behind** — facts learned, things deferred to later
